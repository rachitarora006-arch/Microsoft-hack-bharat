A deep dive into how the Market Intelligence System ingests data, processes analytics, and generates AI narratives in real-time.

⚙️ System Overview & Data Pipeline

The core of the application is a high-performance Python streaming engine designed to process market ticks and news articles concurrently. The data flows through multiple stateful analytical modules before being broadcasted via WebSockets.

🧠 The Pathway Framework — AI Chat & RAG

The interactive AI Assistant on the dashboard utilizes Retrieval-Augmented Generation (RAG). When a user asks a question, the system queries the local vector database for relevant news context, combines it with the live mathematical state of the market, and prompts the LLM to provide a highly contextual and accurate answer.

📂 Project File Architecture

The modular Python backend ensures separation of concerns. Each module is a pure-function processor invoked by the streaming engine.

├── main.py # Application orchestrator, WS handling, and FastAPI routes ├── config.py # Central configuration (API keys, tuning params, thresholds) ├── streaming_engine.py # Custom concurrency engine handling streams and callbacks ├── ingestion.py # Generates simulated ticks and reads news from CSV ├── indicators.py # Computes SMA, EMA, MACD, RSI, Bollinger Bands ├── regime_detection.py # Heuristic-based market state classification ├── anomaly_detection.py # Detects price spikes, volume surges, and flash crashes ├── portfolio_engine.py # Calculates PnL, Value at Risk (VaR), and Risk Scores ├── correlation_engine.py # Computes Pearson correlations between asset pairs ├── rag_module.py # Embeds news articles and runs local FAISS vector search ├── ai_explanations.py # Formats prompts and communicates with OpenAI APIs └── frontend/ # Standard web technologies (HTML, CSS, Vanilla JS)
