🔷 What is the Pathway Framework?

Pathway is an open-source Python framework for real-time data processing and AI pipelines. It allows developers to build streaming applications that process data as it arrives — not in batches. Think of it as a Python-native alternative to Apache Kafka Streams or Apache Flink, but designed specifically for AI/ML workloads.

Core Pathway Concepts

  • pw.Table — An ever-updating table of streaming records. As new data arrives, downstream computations automatically re-execute.
  • ConnectorSubject — A background data producer that feeds records into a pw.Table. You implement run() and call self.next() to emit data.
  • pw.io.csv.read() — Reads CSV files from a directory in streaming mode, watching for new files and emitting new rows automatically.
  • pw.io.python.read() — Creates a table from a Python ConnectorSubject, enabling custom data sources.
  • UDFs (User-Defined Functions) — Stateful or stateless functions decorated with @pw.udf that transform stream records. State is carried between invocations via serialized JSON.
  • JSONL Sinks — Output connectors that write processed results to JSONL files for persistence or debugging.
  • Subscriptions — Callbacks registered on tables that fire whenever new records arrive, enabling event-driven architectures.
How MarketIntel uses Pathway: Our system implements Pathway-compatible abstractions in streaming_engine.py. Every module — from price ingestion to anomaly detection — operates as a streaming processor connected via Pathway-style tables and subscriptions. Data flows continuously through the pipeline, with each module processing records as they arrive.

Why Pathway?

  • Real-time by default — no batch delays; data is processed as it arrives
  • Stateful processing — indicators like RSI, EMA, and MACD require state across ticks; Pathway UDFs handle this natively
  • Python-native — no JVM or Scala required; pure Python with familiar APIs
  • Built for AI — seamless integration with embedding models, vector stores, and LLMs
  • Scalable — designed to handle high-throughput streams with backpressure management

⚙️ System Architecture Overview

The system is organized into four layers, each built on Pathway's streaming primitives:

  • LAYER 1 — Data Ingestion: Two ConnectorSubjects generate market ticks and news articles, feeding them into StreamTables.
  • LAYER 2 — Analytics Pipeline: Subscribed UDFs process each tick through indicators, regime detection, anomaly detection, correlation tracking, and portfolio calculations.
  • LAYER 3 — AI Layer: News articles are embedded via SentenceTransformer (MiniLM) and stored in a FAISS-like vector index. User queries trigger RAG retrieval + LLM generation.
  • LAYER 4 — Presentation: FastAPI serves REST APIs and a WebSocket endpoint. The frontend dashboard connects via WebSocket for real-time updates.

🧠 RAG-Powered AI Assistant

The AI Assistant uses Retrieval-Augmented Generation (RAG) to provide contextual market analysis. When a user asks a question, the system combines relevant news context with live market data before generating an answer.

How RAG Works in MarketIntel

  • Step 1 — Embed: News articles from CSVs are chunked and embedded using SentenceTransformer (MiniLM-L6-v2) into 384-dimensional vectors.
  • Step 2 — Store: Vectors are stored in an in-memory FAISS-like index alongside BM25 keyword indices for hybrid search.
  • Step 3 — Query: User questions are embedded and compared against stored vectors using cosine similarity. BM25 keyword search runs in parallel.
  • Step 4 — Fuse: Results from both searches are combined using Reciprocal Rank Fusion (RRF) for better recall.
  • Step 5 — Augment: Top-K relevant chunks are combined with live market state (prices, indicators, regimes, anomalies).
  • Step 6 — Generate: The augmented context is sent to GPT-4o-mini, which produces a data-backed, contextual answer.

📂 File-by-File Reference

Every Python module in the backend, explained in detail. Each file follows the Pathway streaming pattern of producers, tables, and subscribers.

main.py
The application orchestrator. Initializes FastAPI, creates all streaming tables, wires up subscriptions between modules, starts the streaming engine, and manages WebSocket connections. It also serves HTML pages and exposes REST API endpoints (/api/ask, /api/market-context).
Key responsibilities: Application startup/shutdown lifecycle, route definitions for all pages (homepage, dashboard, docs, terms), WebSocket endpoint (/ws/market), and the main streaming loop that connects ingestion → processing → broadcast.
Core Orchestrator
config.py
Central configuration hub. All tunable parameters live here: asset lists, initial prices (INR), API keys (loaded from .env via python-dotenv), indicator windows (SMA-20, SMA-50, RSI-14), anomaly thresholds, volatility profiles, and tick intervals.
Design principle: Every magic number in the codebase is defined here. Changing SMA_SHORT_WINDOW from 20 to 10 instantly changes indicator behavior across the entire system.
Config
streaming_engine.py
The Pathway-compatible streaming framework. Implements the core streaming abstractions that power the entire system:
StreamTable — Analogous to pw.Table. A thread-safe, observable data stream. Producers emit records via .emit(), and consumers register callbacks via .subscribe(). Supports replay for late-joining subscribers and JSONL sinks for persistence.

ConnectorSubject — Analogous to pw.io.python.ConnectorSubject. A base class for background data producers. Subclass it, implement run(), call self.next() to emit records. Runs in a daemon thread.

CSVDirectoryWatcher — Analogous to pw.io.csv.read(mode="streaming"). Watches a directory for CSV files, emits rows as stream records, tracks processed files to avoid duplicates.

@udf decorator — Marks functions as User-Defined Functions for stream processing. Supports stateful processing where state is serialized as JSON between invocations.

StreamingEngine — The top-level orchestrator that manages all tables, starts/stops all connectors, and provides a unified lifecycle.
Core Pathway
ingestion.py
Data source layer. Provides two streaming data sources using Pathway's connector pattern:
MarketDataGenerator — Generates realistic random-walk price data for BTC, ETH, SOL, NIFTY, SENSEX. Uses volatility profiles from config, with √(interval) scaling for tick-level volatility. Injects random 2-6% spike/drop events at 1% probability for anomaly detection testing.

MarketSubject (ConnectorSubject) — Wraps MarketDataGenerator as a Pathway-style background producer. Calls self.next(symbol, price, volume, timestamp) every 2 seconds.

News Stream — Uses CSVDirectoryWatcher (Pathway's pw.io.csv.read) to watch data/news/ for CSV files. New files are automatically ingested and emitted as stream records.
Data Source Pathway
indicators.py
Stateful technical indicator computation. A single @udf-decorated function (compute_indicators) that maintains rolling windows and computes all indicators incrementally.
Computed per tick: SMA-20, SMA-50, EMA-12, EMA-26, MACD line, MACD signal, MACD histogram, RSI-14, Bollinger Bands (upper/middle/lower), rolling volatility, SMA slope, volume average.

State is carried between invocations via serialized JSON (Pathway UDF pattern). Each symbol has independent state, enabling parallel processing. Window sizes are bounded to prevent unbounded memory growth.
Analytics Pathway UDF
regime_detection.py
Market state classifier. Uses heuristic rules on SMA slopes and volatility to classify each asset into one of four regimes:
📈 BULLISH — SMA-20 is above SMA-50 with a positive slope, indicating upward momentum.
📉 BEARISH — SMA-20 is below SMA-50 with a negative slope, indicating downward pressure.
↔️ SIDEWAYS — SMA slopes are near zero, price is range-bound with no clear direction.
🌊 HIGH_VOLATILITY — Rolling volatility exceeds 2× the configured baseline, indicating extreme price swings.
Analytics
anomaly_detection.py
Statistical anomaly detector. Monitors price and volume streams for three types of anomalies using Z-score analysis:
🔴 PRICE_SPIKE / PRICE_DROP — Triggered when price deviates >2.5 standard deviations from the rolling mean. Severity tiers: LOW (2.5σ), MEDIUM (3.5σ), HIGH (5σ), CRITICAL (>7σ).
📊 VOLUME_SURGE — Triggered when volume exceeds 3× the rolling average. Often indicates institutional activity.
⚡ FLASH_CRASH — Triggered when price drops >5% within 60 seconds. The most severe anomaly type. Each anomaly is broadcast immediately via WebSocket as an alert.
Analytics
correlation_engine.py
Cross-asset correlation tracker. Computes rolling Pearson correlations between all asset pairs (BTC-ETH, BTC-SOL, NIFTY-SENSEX, etc.) using a sliding window of recent prices.
Correlation values range from -1 (perfectly inverse) to +1 (perfectly correlated). This helps identify diversification opportunities and risk concentrations. Updated every tick cycle and displayed on the dashboard.
Analytics
portfolio_engine.py
Portfolio risk management engine. Loads positions from data/portfolio.csv and recalculates risk metrics on every price tick.
Metrics computed: Total portfolio value, PnL (Profit & Loss) in ₹ and %, Value at Risk at 95% confidence (VaR), maximum drawdown percentage, and per-asset risk scores (0-100 composite). VaR uses historical simulation on the rolling return distribution.
Analytics Risk
rag_module.py
Retrieval-Augmented Generation engine. Handles news ingestion, embedding, storage, and hybrid search for the AI Assistant.
Uses SentenceTransformer (MiniLM-L6-v2) to embed text into 384-dimensional vectors. Stores documents in an in-memory vector index with both cosine-similarity search and BM25 keyword search. Results are fused via Reciprocal Rank Fusion (RRF). The retrieve(query) function returns the top-K most relevant news chunks for any question.
AI RAG
ai_explanations.py
AI narration and Q&A engine. The brain of the AI Assistant. Handles three modes of operation:
1. RAG + LLM Mode: Retrieves relevant news via rag_module, combines with live market state, sends to GPT-4o-mini for a contextual answer.
2. Knowledge Base Mode: Routes educational questions (e.g., "what is Bitcoin?") to the comprehensive knowledge_base.py.
3. Template Fallback: For live-data questions when no LLM is available, builds structured answers using real-time indicator data, regime info, and anomalies.

Also generates automated market narratives every 5 minutes summarizing top movers, volatility, regimes, and anomalies.
AI LLM
knowledge_base.py
Comprehensive knowledge repository. Contains 20+ detailed entries covering project architecture, stock market education, crypto fundamentals, and technical analysis — all accessible to the chatbot without needing an LLM.
Categories: Project overview and architecture, NIFTY 50 and SENSEX details, stock market basics, trading and order types, BTC/ETH/SOL deep-dives, DeFi overview, Indian crypto taxation, and technical indicator explanations (RSI, MACD, SMA, Bollinger Bands, etc.)
AI Knowledge
websocket_manager.py
WebSocket connection manager. Thread-safe manager for all active WebSocket connections. Enables real-time broadcasting from background streaming threads to async FastAPI WebSocket endpoints.
Uses asyncio.run_coroutine_threadsafe to bridge the gap between synchronous streaming threads and the async WebSocket layer. Handles client connect/disconnect gracefully, with automatic cleanup of dead connections. Packages all messages with event type and UTC timestamp.
Infrastructure

Risk Score Interpretation

  • 0–30: ✅ Low risk — Conservative portfolio with minimal volatility exposure
  • 30–60: ⚠️ Moderate risk — Balanced exposure, monitor closely during volatile periods
  • 60–100: 🔴 High risk — Significant exposure to volatile assets, consider rebalancing
